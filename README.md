# Threat Monitoring & Response Tool

Инструмент автоматизированного мониторинга и реагирования на угрозы на основе анализа сетевых логов и данных внешних API. Реализован в рамках учебного проекта по программированию на Python (темы 7–14).

---

## Краткое описание

Система выполняет сбор данных из нескольких источников (файлы логов в формате CSV/JSON и API VirusTotal), обогащение и анализ с целью выявления подозрительной активности, имитацию мер реагирования (блокировка IP) и формирование структурированного отчёта с визуализацией. Поддерживаются консольный режим и веб-интерфейс на Streamlit с единой точкой входа.

---

## Функциональность

- **Сбор данных:** загрузка сетевых событий (логи в стиле Suricata: `timestamp`, `src_ip`, `dest_ip`, `event_type`, `severity`). При отсутствии файла логов выполняется автоматическая генерация синтетических данных для демонстрации работы.
- **Обогащение:** проверка уникальных IP-адресов источника через VirusTotal API (эндпоинт проверки IP). При отсутствии API-ключа используется режим имитации на основе локальных JSON-ответов.
- **Детекция угроз:** комбинирование двух критериев — аномально высокая частота запросов от одного IP (порог настраивается, по умолчанию > 100 событий в минуту) и наличие срабатываний в VirusTotal (`last_analysis_stats.malicious` > 0). Результат анализа помечается полем `is_threat` и причиной срабатывания.
- **Реагирование:** для каждой выявленной угрозы формируется запись в лог в формате `[ALERT]`, выполняется имитация блокировки (добавление IP в список и запись в файл `blocked_ips.txt`).
- **Отчётность:** сохранение полного результата анализа в форматах CSV и JSON (включая метаданные: дата генерации, счётчики событий и угроз), построение графика «Top-5 самых активных IP» с выделением угроз (PNG).
- **Веб-дашборд:** просмотр сводных метрик, таблицы отчёта с фильтрацией, графика, списка заблокированных IP и хвоста лога; запуск полного цикла анализа из интерфейса с пошаговым отображением этапов.

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.8+ |
| HTTP-клиент, API | `requests` |
| Обработка данных | `pandas` |
| Визуализация | `matplotlib`, `seaborn` |
| Конфигурация | `python-dotenv`, стандартный модуль `json` |
| Веб-интерфейс | `streamlit` |

Типизация соответствует PEP 484; логирование выполняется через стандартный модуль `logging`.

## Структура проекта

```
├── main.py              # Ядро пайплайна: загрузка логов, обогащение, детекция, реагирование, отчёт, график
├── app.py               # Streamlit-приложение: дашборд и запуск анализа
├── run.py               # Единая точка входа (запуск веб-интерфейса)
├── requirements.txt     # Зависимости
├── .env.example         # Шаблон переменных окружения
├── .streamlit/          # Конфигурация темы и клиента Streamlit
├── data/                # Входные логи (CSV/JSON) и кэш ответов API (mock_api/)
└── output/              # Результаты: отчёты (CSV, JSON), график (PNG), blocked_ips.txt, app.log
```

## Требования и запуск

**Требования:** Python 3.8 или новее, зависимости из `requirements.txt`.

Установка зависимостей:

```bash
pip install -r requirements.txt
```

**Рекомендуемый способ запуска** — веб-интерфейс (единая точка входа):

```bash
python run.py
```

Откроется браузер с дашбордом; при необходимости анализ запускается кнопкой в боковой панели.

**Консольный режим** (без веб-интерфейса):

```bash
python main.py
```

Результаты записываются в каталог `output/`: `threat_report.csv`, `threat_report.json`, `threat_plot.png`, `blocked_ips.txt`, `app.log`.

---

## Конфигурация

Опционально в корне проекта создаётся файл `.env` (по образцу `.env.example`). Переменная `VIRUSTOTAL_API_KEY` задаёт ключ для доступа к VirusTotal API; при её отсутствии используется режим имитации с локальными ответами в `data/mock_api/`, что позволяет запускать проект без регистрации на внешних сервисах.

Документация по получению API-ключа: [VirusTotal — регистрация](https://www.virustotal.com/gui/join-us).
